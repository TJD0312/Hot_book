# Skills 到底是什么？

## 选题定位

本篇聚焦 `Skills` 的入门理解，核心问题是：`Skills` 到底是什么？

面向已经听过 `Agent`、`Tools` 或 `MCP`，但还不清楚 `Skills` 位置的普通读者。本文不从某个平台的具体实现讲起，而是把 `Skills` 解释成 Agent 可以复用的一套“做事方法包”：它更像“操作手册 + 示例 + 约束”，告诉 Agent 遇到某类任务时应该按什么流程处理、参考什么经验、遵守哪些边界。

本篇重点和 `MCP` 区分：`MCP` 更偏“怎么连接外部能力”，`Skills` 更偏“连接之后，或面对某类任务时，应该怎么把事情做好”。

## 核心观点

`Skills` 可以理解为给 Agent 准备的可复用任务方法包。

一个 `Skill` 通常不是工具本身，也不是连接工具的协议。它更像一份面向 Agent 的操作说明：什么时候应该使用、应该按什么步骤推进、有哪些参考示例、哪些行为不能做、遇到风险时该如何处理。

如果说 `MCP` 解决的是“AI 应用怎样连接外部工具、数据和服务”，那么 `Skills` 更像解决“Agent 遇到某类任务时，怎样沿用一套经过整理的方法去做”。前者偏连接，后者偏流程和经验。

可以用一个边界清晰的类比：

- `MCP` 像插口和连接规则。
- `Tools` 像可以被调用的外部能力。
- `Skills` 像教 Agent 如何完成某类任务的操作手册。

这个类比不是严格定义，只用于帮助非技术读者理解三者分工。

## 表达边界

- 不把 `Skills` 说成统一行业协议。不同平台、系统或项目对 `Skills` 的定义和实现可能不同。
- 不把 `Skills` 等同于 `MCP`。`MCP` 更偏连接外部工具、数据和服务的协议；`Skills` 更偏任务方法、流程说明和经验复用。
- 不把 `Skills` 等同于 `Tools`。`Tools` 是可调用能力，`Skills` 是指导 Agent 如何完成任务的方法说明。
- 不承诺有了 `Skills`，Agent 就一定不会出错。错误仍可能来自模型理解偏差、工具结果不可靠、示例不充分、约束写得不清楚或任务本身变化。
- 不引入具体平台的最新功能、价格、API 参数或产品能力，避免未经核查的时效性描述。
