# 标题页

标题：看懂 AI 模型成绩单

副标题：常见 benchmark 速查表

封面辅助文案：先看它在考哪门课，再看分数高不高

# 第 1 页

页面标题：benchmark 不是总分，是分科考试

页面正文：

模型发布页里的成绩单，通常不是一个统一总分。

它是一张分科成绩表：有的考知识，有的考数学，有的考代码，有的考读图、浏览网页、操作电脑和调用工具。

所以读成绩单时，第一步不是看谁最高，而是看它到底在考什么。

# 第 2 页

页面标题：知识与专家题：看模型懂多少、推多深

页面正文：

`MMLU`：综合学科笔试，覆盖多门学科知识。

`MMLU-Pro`：更难版本，减少简单题，增加推理要求。

`GPQA`：博士级理科难题，重点看专业知识和复杂推理。

`Humanity's Last Exam`：专家级综合难题，用来拉开前沿模型差距。

# 第 3 页

页面标题：数学推理：看它能不能真正算明白

页面正文：

`AIME`：常见数学竞赛题，适合观察模型的多步推理和计算能力。

`FrontierMath`：专家级现代数学难题，通常比普通竞赛题更难。

这类分数高，说明模型在数学推理上更强；但不代表它在写作、办公或客服场景里也一定更好用。

# 第 4 页

页面标题：代码能力：从写函数到修真实项目

页面正文：

`HumanEval`：看模型能不能根据说明写出短小函数。

`SWE-bench`：看模型能不能修真实 GitHub issue。

`Terminal-Bench`：看模型能不能在命令行里完成多步任务。

代码 benchmark 越往后，越接近真实工程环境，也越考验长期规划和工具使用。

# 第 5 页

页面标题：多模态理解：不是看见图片就够了

页面正文：

`MMMU`：看模型能不能结合图片、图表和专业知识答题。

`MathVista`：看模型能不能在视觉场景中做数学推理。

多模态成绩高，不只是“识图能力强”，还要能把看到的信息用于推理和解题。

# 第 6 页

页面标题：网页与电脑操作：看它会不会找资料、用界面

页面正文：

`BrowseComp`：看模型能不能持续浏览网页，找到难查的信息。

`OSWorld`：看模型能不能操作真实电脑环境，完成跨应用任务。

这类 benchmark 更接近 Agent 能力：模型不只是回答问题，还要在环境里行动。

# 第 7 页

页面标题：Agent 工具流程：看它能不能连续做事

页面正文：

`Toolathlon`：看模型能不能跨多个软件和工具完成长流程任务。

`TAU-bench`：看模型能不能在对话中使用工具，并遵守业务规则。

这类评测关注的不是单次回答，而是多轮执行、工具协调和任务稳定性。

# 第 8 页

页面标题：真实工作与安全任务：更接近高风险场景

页面正文：

`GDPval`：看模型能不能完成接近真实知识工作的交付物，例如分析、文档、表格和计划。

`CyberGym`：看模型在网络安全任务中的能力表现。

这类成绩更贴近实际应用，但也更需要看评测范围、限制条件和安全边界。

# 第 9 页

页面标题：读成绩单，先问三个问题

页面正文：

第一，它考的是哪类能力：知识、数学、代码、图像、工具，还是真实工作？

第二，它是在固定题库里答题，还是要操作环境、调用工具、完成交付物？

第三，它和你的任务有多接近？

看懂这三点，分数才有真正的参考价值。

# 小红书正文文案

正文标题：看懂 AI 模型成绩单：常见 benchmark 速查表

正文内容：

🧩 很多模型发布时都会放一张“成绩单”：`MMLU`、`GPQA`、`SWE-bench`、`OSWorld`、`BrowseComp`、`GDPval`……分数一个比一个高，但如果不知道这些项目在考什么，就很容易把它们都看成“模型总分”。

💡 更准确的理解是：`benchmark` 不是统一总分，而是一组分科考试。不同项目测的是不同能力。有的像综合知识笔试，有的像数学竞赛，有的像真实代码修复，有的要求模型浏览网页、操作电脑、调用工具，甚至产出接近真实工作的交付物。

📚 例如 `MMLU` 和 `MMLU-Pro` 主要看多学科知识与推理；`GPQA` 和 `Humanity's Last Exam` 更偏专家级难题。它们能说明模型在知识问答和复杂推理上的水平，但不能直接代表它在所有真实任务里都好用。

🧮 数学类常见 `AIME` 和 `FrontierMath`。前者常用于观察竞赛数学推理，后者更偏专家级现代数学。数学分数高，说明模型在这类题上更强，但不等于它一定更会写方案、做客服或处理办公流程。

🧑‍💻 代码类也要分层看。`HumanEval` 更像“按说明写短函数”；`SWE-bench` 更像“修真实项目里的 GitHub issue”；`Terminal-Bench` 则更强调在命令行环境里连续完成任务。越接近真实工程，越考验规划、调试和工具使用。

🖼️ 多模态类常见 `MMMU` 和 `MathVista`。它们不是只看模型能不能识别图片，而是看模型能不能把图表、公式、示意图里的信息用于推理。看到这类分数时，要想到“读图 + 学科理解 + 推理”。

🧭 Agent 类 benchmark 这几年越来越常见。`BrowseComp` 看模型能不能持续浏览网页找难查信息；`OSWorld` 看它能不能操作真实电脑环境；`Toolathlon` 和 `TAU-bench` 更关注跨工具、多轮对话和业务规则下的连续执行。

📋 还有一些更贴近真实工作或高风险场景的项目。`GDPval` 看模型能不能完成知识工作交付物，例如分析、文档、表格和计划；`CyberGym` 则关注网络安全任务能力。看到这类分数时，更要看评测范围和安全边界。

✅ 所以，下次看到模型成绩单，可以先问三个问题：它考的是哪类能力？它是在答题，还是在操作环境、调用工具、完成交付物？它和我的真实任务有多接近？先看懂“考哪门课”，分数才有参考价值。
