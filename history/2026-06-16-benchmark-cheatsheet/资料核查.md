# 资料核查

核查日期：2026-06-16

## 本篇采用的核心判断

- 主流模型厂商发布模型时，经常用多项 `benchmark` 展示模型能力，而不是只展示一个总分。
- 不同 `benchmark` 对应不同能力维度，包括学科知识、专家推理、数学、代码、多模态、网页浏览、电脑操作、工具调用、真实知识工作和网络安全任务。
- 读模型成绩单时，应先判断某项 `benchmark` 在测什么，再理解分数高低。
- 本篇不引用具体模型实时排名，不比较具体模型优劣，不把某个单项分数写成模型整体能力结论。

## 厂商发布材料核查

1. OpenAI：*Introducing GPT-5.5*  
   链接：https://openai.com/index/introducing-gpt-5-5/  
   核查要点：该发布页的评测表中列出 `Terminal-Bench 2.0`、`Expert-SWE (Internal)`、`GDPval`、`OSWorld-Verified`、`Toolathlon`、`BrowseComp`、`FrontierMath`、`CyberGym` 等项目。正文也解释了部分项目，例如 `Terminal-Bench 2.0` 用于复杂命令行工作流，`GDPval` 用于知识工作任务，`OSWorld-Verified` 用于真实电脑环境操作。

2. Anthropic：*Introducing Claude 4*  
   链接：https://www.anthropic.com/news/claude-4  
   核查要点：该发布页强调 `SWE-bench`、`Terminal-bench` 等代码与 Agent 相关评测，并在附录中列出 `TAU-bench`、`GPQA Diamond`、`MMMLU`、`MMMU`、`AIME` 等评测项目和报告方式。

3. Google：*Gemini 2.5: Our most intelligent AI model*  
   链接：https://blog.google/innovation-and-ai/models-and-research/google-deepmind/gemini-model-thinking-updates-march-2025/  
   核查要点：该发布页提到 `LMArena`、`GPQA`、`AIME 2025`、`Humanity's Last Exam`、`SWE-Bench Verified` 等项目，用于展示模型在人类偏好、推理、数学、专家知识和代码任务上的表现。

## benchmark 原始资料与定义核查

1. `MMLU`：Dan Hendrycks 等，*Measuring Massive Multitask Language Understanding*  
   链接：https://arxiv.org/abs/2009.03300  
   核查要点：该测试覆盖 57 个任务，用于衡量文本模型的多任务准确率、世界知识和问题解决能力。

2. `MMLU-Pro`：Yubo Wang 等，*MMLU-Pro: A More Robust and Challenging Multi-Task Language Understanding Benchmark*  
   链接：https://arxiv.org/abs/2406.01574  
   核查要点：该工作在 `MMLU` 基础上增强难度，加入更复杂的推理问题，并将选项数量从 4 个扩展到 10 个。

3. `GPQA`：David Rein 等，*GPQA: A Graduate-Level Google-Proof Q&A Benchmark*  
   链接：https://arxiv.org/abs/2311.12022  
   核查要点：该数据集包含由领域专家撰写的生物、物理、化学选择题，难度接近研究生级别，并强调普通网络搜索难以直接解决。

4. `Humanity's Last Exam`：Long Phan 等，*Humanity's Last Exam*  
   链接：https://arxiv.org/abs/2501.14249  
   核查要点：该评测包含跨学科专家级题目，用于衡量模型在前沿人类知识和推理问题上的表现。

5. `AIME`  
   核查要点：本文只使用厂商发布材料中将 `AIME` 作为数学推理评测项这一事实，不展开具体竞赛赛制。具体赛制如需写入图片或正文，需要补充美国数学竞赛官方来源核查。

6. `FrontierMath`：Elliot Glazer 等，*FrontierMath: A Benchmark for Evaluating Advanced Mathematical Reasoning in AI*  
   链接：https://arxiv.org/abs/2411.04872  
   核查要点：该评测由专家数学家设计和审校，包含现代数学中的高难原创题，用于衡量高级数学推理能力。

7. `HumanEval`：Mark Chen 等，*Evaluating Large Language Models Trained on Code*  
   链接：https://arxiv.org/abs/2107.03374  
   核查要点：该工作提出 `HumanEval`，用于测试模型根据自然语言描述生成 Python 函数的能力。

8. `SWE-bench`：Carlos E. Jimenez 等，*SWE-bench: Can Language Models Resolve Real-World GitHub Issues?*  
   链接：https://arxiv.org/abs/2310.06770  
   核查要点：该评测由真实 GitHub issue 和对应 pull request 构成，要求模型修改代码库来解决问题。

9. `Terminal-Bench`：Mike A. Merrill 等，*Terminal-Bench: Benchmarking Agents on Hard, Realistic Tasks in Command Line Interfaces*  
   链接：https://arxiv.org/abs/2601.11868  
   核查要点：该评测由命令行环境中的高难真实任务构成，用于测试 Agent 在终端里规划、执行和验证任务的能力。

10. `MMMU`：Xiang Yue 等，*MMMU: A Massive Multi-discipline Multimodal Understanding and Reasoning Benchmark for Expert AGI*  
    链接：https://arxiv.org/abs/2311.16502  
    核查要点：该评测包含大学考试、测验和教材中的多模态问题，覆盖图表、地图、表格、化学结构等多种图像类型。

11. `MathVista`：Pan Lu 等，*MathVista: Evaluating Mathematical Reasoning of Foundation Models in Visual Contexts*  
    链接：https://arxiv.org/abs/2310.02255  
    核查要点：该评测关注视觉场景中的数学推理，要求模型结合图像理解和组合推理。

12. `BrowseComp`：Jason Wei 等，*BrowseComp: A Simple Yet Challenging Benchmark for Browsing Agents*  
    链接：https://arxiv.org/abs/2504.12516  
    核查要点：该评测包含需要持续浏览互联网才能找到答案的问题，用于衡量 Agent 的网页浏览和信息查找能力。

13. `OSWorld`：Tianbao Xie 等，*OSWorld: Benchmarking Multimodal Agents for Open-Ended Tasks in Real Computer Environments*  
    链接：https://arxiv.org/abs/2404.07972  
    核查要点：该评测使用真实电脑环境和真实应用，评估多模态 Agent 完成开放式电脑任务的能力。

14. `Toolathlon`：Junlong Li 等，*The Tool Decathlon: Benchmarking Language Agents for Diverse, Realistic, and Long-Horizon Task Execution*  
    链接：https://arxiv.org/abs/2510.25726  
    核查要点：该评测覆盖多个软件应用和工具，关注语言 Agent 在真实、多样、长流程任务中的执行能力。

15. `TAU-bench`：Shunyu Yao 等，*$\tau$-bench: A Benchmark for Tool-Agent-User Interaction in Real-World Domains*  
    链接：https://arxiv.org/abs/2406.12045  
    核查要点：该评测模拟用户、工具和业务规则之间的交互，用于评估 Agent 在真实领域中使用工具并遵守规则的稳定性。

16. `GDPval`：Tejal Patwardhan 等，*GDPval: Evaluating AI Model Performance on Real-World Economically Valuable Tasks*  
    链接：https://arxiv.org/abs/2510.04374  
    核查要点：该评测覆盖 44 个职业中的真实知识工作任务，关注模型产出交付物的质量。

17. `CyberGym`：Zhun Wang 等，*CyberGym: Evaluating AI Agents' Cybersecurity Capabilities with Real-World Vulnerabilities at Scale*  
    链接：https://arxiv.org/abs/2506.02548  
    核查要点：该评测基于真实软件漏洞，关注 AI Agent 在网络安全任务中的能力表现。本文只做能力评测解释，不提供任何攻击步骤或操作建议。

## 本篇不采用的内容

- 不采用第三方实时榜单排名作为正文结论。
- 不引用截图中的具体百分比作为图片正文，因为模型分数更新快，容易过时。
- 不展开 `Expert-SWE (Internal)`，因为它是厂商内部评测，普通读者难以从公开资料中完整理解其题目和评测细节。
- 不用某个 benchmark 分数推断“模型整体最好”。
