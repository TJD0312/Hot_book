# 资料来源与核查记录

## 本次核查结论

- `RAG`（Retrieval-Augmented Generation，检索增强生成）的关键机制是把语言模型与外部检索机制结合，让模型在生成回答时参考被检索出的资料。
- 本文将 `RAG` 类比为“开卷考试”，用于说明它更偏向“提供外部资料”，而不是直接修改模型参数。
- 微调更适合通过训练样例调整模型参数和模型行为，例如输出格式、语气、分类方式、任务流程和特定任务表现。
- `RAG` 和微调可以配合使用，但不能混为一谈：一个偏向“检索和上下文补充”，一个偏向“训练、参数调整和行为调整”。

## 参考来源

1. Patrick Lewis et al., 2020, *Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks*  
   来源：arXiv  
   链接：https://arxiv.org/abs/2005.11401  
   核查点：论文提出 `RAG` 模型，将预训练模型的参数化记忆与外部非参数化记忆结合；外部记忆由检索器访问。

2. OpenAI API Docs, *Retrieval*  
   来源：OpenAI Developers  
   链接：https://developers.openai.com/api/docs/guides/retrieval  
   核查点：检索用于对数据做语义搜索，并可与模型结合生成回答；外部资料通常通过向量库、检索结果和上下文进入模型回答流程。

3. OpenAI API Docs, *Model optimization*  
   来源：OpenAI Developers  
   链接：https://developers.openai.com/api/docs/guides/model-optimization  
   核查点：模型优化包括 evals、prompt engineering 和 fine-tuning；文档说明可以在提示中提供相关上下文，也说明微调通过输入输出样例让模型更擅长某类任务。

## 表达边界

- “模型变聪明”是口语化说法，本文用它指代“模型参数本身获得了新能力或新知识”。严格说，`RAG` 可能提升最终系统回答质量，但不等于基础模型参数被更新。
- “开卷考试”只是类比，不代表 `RAG` 的全部工程细节。真实系统还涉及资料切分、向量检索、重排、上下文拼接、引用和评估。
- 微调会调整模型参数，但并不是万能的知识注入方式。对于频繁更新、需要溯源、资料量大或事实要求高的场景，通常更适合优先考虑 `RAG` 或 `RAG` 与微调配合。
