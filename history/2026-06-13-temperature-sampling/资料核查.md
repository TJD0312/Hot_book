# 资料核查

## 已确认的通用解释

- `LLM`（大语言模型）生成文本时，通常会基于当前上下文对后续 `Token`（词元，文本切分后的基本单位）形成概率分布。
- 采样类解码会从候选 `Token` 中选择一个继续生成，因此同一输入在不同运行中可能产生不同输出。
- `temperature`（采样随机性参数）通常用于调整概率分布的平滑程度：较低时更偏向高概率候选，较高时会提高多样候选被选中的机会。
- 从用户体验上看，较低 `temperature` 往往更稳定，较高 `temperature` 往往更多样，但它不是事实正确率或模型能力的直接开关。

## 本篇刻意不展开的内容

- 不讨论具体平台或 API 的 `temperature` 默认值、上限、下限和推荐数值。
- 不讨论 `top_p`、`top_k`、beam search 等其他解码参数，以免单篇内容过载。
- 不把回答差异完全归因于 `temperature`，因为模型版本、上下文、系统提示词、检索结果、工具调用和服务端策略也可能影响输出。
