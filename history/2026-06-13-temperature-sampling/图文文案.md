# 标题页

标题：同一条 Prompt，为什么每次答案不一样？

副标题：关键常在 temperature

封面辅助文案：稳定回答和创意回答，不是同一种设置

# 第 1 页

页面标题：同一条提示词，不一定只有一个答案

页面正文：  
你问 `LLM` 同一个问题，它可能第一次写得很简洁，第二次换了例子，第三次又换了表达。

这并不一定是模型“忽然变聪明”或“忽然变笨”。

很多时候，是因为生成回答时，模型会在多个可能的续写里做选择。

# 第 2 页

页面标题：temperature 调的是“随机性”

页面正文：  
`temperature` 是采样随机性参数。

它影响模型在生成时有多“保守”或多“发散”。

低 `temperature`：更常选择概率最高的表达。  
高 `temperature`：让更多候选表达也有机会出现。

所以它改变的不是问题本身，而是模型选择答案的方式。

# 第 3 页

页面标题：想要稳定，就把随机性调低

页面正文：  
低 `temperature` 的作用是：

“优先选最稳、最常见、最像标准答案的说法。”

它适合这类任务：

提取信息、分类判断、固定格式输出、总结事实、生成结构化结果。

你希望每次结果都接近一致时，稳定比新鲜更重要。

# 第 4 页

页面标题：想要创意，就允许它多试几种说法

页面正文：  
高 `temperature` 的作用是：

“不要只盯着最常见答案，也可以试试别的合理表达。”

它适合这类任务：

起标题、写广告语、做头脑风暴、改写语气、寻找不同表达角度。

你希望看到更多可能性时，多样性比一致性更重要。

# 第 5 页

页面标题：可以把它想成“候选答案抽签”

页面正文：  
模型并不是每一步都只有一个选项。

它会看到一组候选续写：有的概率很高，有的概率较低但也说得通。

低 `temperature`：几乎总是抽到最稳的那张。  
高 `temperature`：其他候选也更容易被抽中。

这个类比只帮助理解随机性，不代表模型真的在纸箱里抽签。

# 第 6 页

页面标题：稳定回答，不等于一定正确

页面正文：  
低 `temperature` 会让回答更一致，但不保证它更真实。

如果问题本身模糊、上下文不足，或者模型掌握的信息不可靠，它可能会稳定地给出同一种错误答案。

所以事实类问题不能只靠“调低随机性”。

更可靠的做法是：补充资料、要求来源、交叉核查。

# 第 7 页

页面标题：创意回答，也不等于随便乱答

页面正文：  
高 `temperature` 会增加变化，但它不应该被理解成“让模型乱写”。

如果你给出清楚边界，它依然可以在范围内发散：

目标读者是谁？  
语气要正式还是轻松？  
不能偏离哪些事实？  
需要几个备选方案？

创意需要空间，也需要约束。

# 第 8 页

页面标题：真正要记住的事

页面正文：  
`temperature` 不是“聪明按钮”，而是“回答多样性按钮”。

想要稳定、可复查、格式一致：用低一点的随机性。

想要灵感、改写、不同角度：允许更高的随机性。

重要事实问题，还要配合资料来源和人工核查。

# 小红书正文文案

正文标题：同一条 Prompt，为什么 LLM 每次答案不一样？

正文内容：
🧩 同一条 `Prompt` 每次答案不一样，很多时候不是模型忽然变聪明或变笨，而是生成时存在采样。`temperature` 调的就是这种选择范围的“发散程度”。

🌡️ 低 `temperature` 会更偏向概率最高、更稳妥的表达。它适合信息抽取、分类、事实总结和固定格式输出，因为这些任务更需要一致、可复查。

🎨 高 `temperature` 会让更多候选表达有机会出现。起标题、写文案、头脑风暴、改写语气时，你需要的往往不是唯一答案，而是不同角度。

📋 这也解释了“同问不同答”。模型不是从固定答案库里复制，而是在已有上下文和候选 `Token` 分布中继续生成。采样越开放，表达变化通常越明显。

🖼️ 可以把它想成候选答案抽签：低随机性总抽最稳那张，高随机性允许其他合理选项出现。但类比只帮助理解，模型并不是真的在纸箱里抽纸条。

✅ 别把 `temperature` 当“聪明按钮”。事实类问题要补资料、看来源、做核查；创意类任务也要给边界。参数能改变表达发散度，不能替代任务设计和事实校验。
