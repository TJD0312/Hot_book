# 同一条 Prompt，为什么 LLM 每次回答都不一样？

## 选题定位

本篇聚焦 `temperature`（采样随机性参数），解释为什么同一条 `Prompt`（给模型的输入指令）多次提交后，`LLM`（大语言模型）可能给出不完全相同的回答。

核心不是泛泛讲 `LLM` 如何生成文本，而是讲清楚：模型生成时通常不是只有一个“唯一正确的下一个词”，而是在多个可能的续写中做选择；`temperature` 会影响这个选择过程的随机性。

## 核心观点

同一条提示词每次回答不一样，很多时候不是模型“故意变来变去”，而是因为生成过程里存在采样选择。

可以把它理解为：模型会给很多可能的下一个 `Token` 打分，形成一个概率分布。`temperature` 会影响模型在这个分布里“多保守”或“多发散”地选择。

- `temperature` 低：更倾向选择概率最高的表达，输出更稳定、更一致。
- `temperature` 高：较低概率但仍合理的表达也更有机会被选中，输出更多样、更有创意。

但要强调边界：`temperature` 不是“正确率按钮”，也不是“聪明程度按钮”。低温度不保证事实正确，高温度也不必然等于乱答。它主要影响生成时的随机性和多样性。

## 表达边界

- 不把 `temperature` 说成唯一影响回答差异的因素；实际差异还可能来自模型版本、系统提示词、上下文、检索结果、工具调用、服务端策略等。
- 不承诺某个具体数值一定适合所有平台，因为不同产品和 API 的取值范围、默认值和实现细节可能不同。
- 不说低 `temperature` 一定正确；它只是更倾向稳定选择。
- 不说高 `temperature` 一定胡说；它只是让更多候选表达有机会出现。
- 不引入具体平台的最新参数默认值，避免未经核查的产品细节。
