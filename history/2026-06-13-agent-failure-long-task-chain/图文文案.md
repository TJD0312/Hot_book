# 标题页

标题：
Agent 为什么会失败？

副标题：
不是模型笨，而是任务链太长

封面辅助文案：
每一步小错，都会在后面被放大

# 第 1 页

页面标题：
Agent 不只是回答一句话

页面正文：
`Agent`（智能体）通常不是只生成一段文字。

它更像是在循环做事：

先理解目标，
再拆成步骤，
调用工具，
观察结果，
然后决定下一步。

任务越长，步骤越多。

步骤越多，出错点也越多。

# 第 2 页

页面标题：
真正危险的是“前错后继”

页面正文：
很多时候，Agent 不是突然在某一步崩掉。

它可能每一步看起来都还算合理：

读需求、拆任务、找资料、调工具、整理结果。

但如果第一步理解偏了一点，
后面的步骤就会沿着这个偏差继续往前走。

前一步的小错，
会变成后一步的输入。

# 第 3 页

页面标题：
小错是这样被放大的

页面正文：
比如你让 Agent 做一份资料整理。

第一步，它误解了你的目标。

第二步，它按错误目标去搜资料。

第三步，它用不完整资料做判断。

第四步，它把中间判断写成总结。

最后的结果可能很完整，
但方向已经偏了。

# 第 4 页

页面标题：
这就是多步任务误差累积

页面正文：
`LLM`（大语言模型）每一步都可能有一点不确定性。

可能是理解偏差，
可能是检索遗漏，
可能是工具参数写错，
也可能是对结果解释过度。

单看某一步，错误很小。

但在长任务里，
小错会被下一步当成事实继续使用。

# 第 5 页

页面标题：
所以 Agent 需要记住“状态”

页面正文：
这里说的 `Memory`（记忆机制），
不是让模型真的像人一样拥有记忆。

更重要的是记录关键状态：

目标是什么？
已经做过什么？
哪些假设还没确认？
哪里可能有风险？

否则任务做得越久，
Agent 越容易忘记一开始要解决什么。

# 第 6 页

页面标题：
校验，是给长任务加闸门

页面正文：
可靠的 Agent 不能只顾着往前走。

它需要 `Verification`（校验机制）：

当前步骤是否符合目标？
工具返回是否可信？
中间结论有没有证据？
输出是否满足约束？

最好不是等最后才检查，
而是在关键节点就拦住错误。

# 第 7 页

页面标题：
回滚，让 Agent 能退回重来

页面正文：
如果发现方向错了，
继续往前做只会增加成本。

`Rollback`（回滚机制）的作用，
就是让系统回到上一个可靠状态。

重新拆任务，
重新调用工具，
或者换一条策略。

它不等于让模型变聪明，
而是让系统不必一直错下去。

# 第 8 页

页面标题：
一句话总结

页面正文：
Agent 做长任务会失败，
常常不是因为模型“太笨”。

而是因为任务链太长，
状态太多，
每一步的小错都可能传给下一步。

所以靠谱的 Agent，
不只是会执行。

还要会记住关键状态，
会及时校验，
也能在必要时回滚。

# 小红书正文文案

正文标题：
Agent 为什么会失败？不是模型笨，而是任务链太长

正文内容：
很多人看到 Agent 做长任务失败，会直接归因于“模型不够聪明”。

但在真实任务里，问题往往没这么简单。

`Agent`（智能体）不是只回答一句话。它通常要先理解目标，再拆成步骤，调用工具，观察结果，然后继续决定下一步。

这类多步任务最容易出现的问题，是“误差累积”。

第一步目标理解偏了一点，第二步就可能按错误目标去找资料；资料不完整，后面的判断就会继续偏；中间判断偏了，最后生成的报告可能看起来很完整，但方向已经错了。

所以，Agent 失败不一定是因为 `LLM`（大语言模型）单步能力太差，也可能是任务链太长，前面的小错被后面不断继承和放大。

这也是为什么可靠的 Agent 系统需要三类机制：

第一是 `Memory`（记忆机制）。它不是让模型真的像人一样拥有记忆，而是记录目标、进度、假设、中间结果和风险点。

第二是 `Verification`（校验机制）。在关键步骤检查：当前动作是否符合目标？工具返回是否可信？中间结论有没有证据？

第三是 `Rollback`（回滚机制）。一旦发现方向错了，就退回上一个可靠状态，重新拆任务或换策略，而不是继续沿着错误路线往前做。

一句话总结：

Agent 做长任务，难点不只是“下一步要不要答对”，而是“很多步之后还能不能不偏航”。

靠谱的 Agent，不只是会执行，还要会记住、会检查、会退回。
