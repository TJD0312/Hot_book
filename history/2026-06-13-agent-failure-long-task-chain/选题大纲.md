# Agent 为什么会失败？不是模型笨，而是任务链太长

## 选题定位

本篇聚焦 `Agent`（智能体）在多步任务中容易失败的原因：很多失败并不是因为 `LLM`（大语言模型）单步能力太差，而是任务链过长之后，前面的小偏差会被后续步骤继续继承、放大，最后变成看起来完整但方向错误的结果。

核心切口是“多步任务误差累积”。这篇不重点讲 `Agent Memory`（智能体记忆）本身是什么，而是解释为什么长链路任务需要记忆、校验和回滚机制。

## 核心观点

`Agent` 失败常常不是某一步特别离谱，而是很多小偏差沿着任务链传递：

1. 目标理解偏了一点。
2. 子任务拆分偏了一点。
3. 检索资料不完整。
4. 工具调用参数写错。
5. 对返回结果解释过度。
6. 最后把中间错误整理成一个完整答案。

因此，可靠的 `Agent` 系统不能只依赖“模型下一步猜得对”。它还需要：

- `Memory`：记录关键目标、状态、假设和中间结果，避免走着走着忘了起点。
- `Verification`：在关键步骤做校验，及时发现输入、工具结果或中间结论是否不可靠。
- `Rollback`：发现路线错误后，能退回上一个可靠状态，换策略重试，而不是继续沿着错误方向前进。

一句话总结：多步任务越长，单步小错越容易累积；好 Agent 不只是会往前做，还要会记住、会检查、会退回。

## 表达边界

- 不把 Agent 失败简单归因于“模型笨”或“模型没能力”。
- 不承诺记忆、校验、回滚一定能消除失败；它们只是降低长链路风险的工程机制。
- 不把 `Memory` 讲成模型本体真的拥有长期记忆。本篇中的记忆主要指系统记录和回填关键状态。
- 不把“误差累积”讲成严格数学结论或特定产品数据；它是多步系统中常见的工程风险表达。
- 不引入具体平台能力、价格或最新模型功能，避免未经核查的产品描述。
