# 标题页

标题：
LLM 不是在“思考”

副标题：
底层动作是预测下一个 Token

封面辅助文案：
AI 像懂了，其实是在一步步续写

# 第 1 页

页面标题：
先说清：Token 不是一句话

页面正文：
`Token` 是模型处理文本时的基本小块。

它可能是一个字、一个词的一部分、一个标点，也可能是一段常见字符组合。

`next token prediction` 不是一次预测“下一句话”，而是一次预测“下一个小文本块”。

# 第 2 页

页面标题：
LLM 每一步都在问同一个问题

页面正文：
给定前面的上下文，模型会计算：

“接下来最可能出现哪个 Token？”

它选出一个 Token 后，把这个 Token 加回上下文，再继续预测下一个。

一段回答，就是这样一步一步生成出来的。

# 第 3 页

页面标题：
可以把它想成“超强续写”

页面正文：
有点像输入法联想：

你输入“今天天气”，它可能推荐“很好”。

但 `LLM` 的规模大得多。它不仅看前几个字，还会参考整段上下文、问题意图、常见表达、知识模式和任务格式。

类比只是帮助理解：它不是普通输入法，而是复杂得多的语言生成模型。

# 第 4 页

页面标题：
为什么它看起来像在推理？

页面正文：
因为训练材料里有大量“人类如何解释问题”的文本：

问答、教程、论文、代码、推导过程、评论、总结……

当你问一个问题时，模型会生成一串很像“解释过程”的 Token。

这会让我们感觉它在思考，但底层动作仍然是连续预测。

# 第 5 页

页面标题：
“像懂了”不等于“真的懂了”

页面正文：
人理解一句话，通常会连接经验、目的、现实反馈和长期记忆。

`LLM` 生成一句话，主要依赖训练中学到的模式，以及当前对话给出的上下文。

它可以表现得很像理解，但不能把每次流畅回答都当成“它真的知道”。

# 第 6 页

页面标题：
这也解释了为什么会“幻觉”

页面正文：
模型追求的是“在当前上下文里接得像”，不是天然保证“每句话都真实”。

当上下文不足、问题有陷阱，或资料本身混乱时，它可能生成很顺口但不准确的内容。

这就是很多 `AI hallucination`（AI 幻觉）的来源：语言上合理，不代表事实一定可靠。

# 第 7 页

页面标题：
真正需要记住的事

页面正文：
`LLM` 的底层动作是：

看上下文，预测下一个 Token，再继续往下接。

所以它很适合帮你起草、改写、总结、解释。

但涉及事实、数据、医学、法律、金融和重要决策时，一定要让它给来源，并自己核查。

# 小红书正文文案

正文标题：
LLM 不是在“思考”，底层动作是预测下一个 Token

正文内容：
🧠 很多人第一次用 `LLM`，都会有一种感觉：

它是不是“真的懂了”？

先看底层动作。

`LLM` 不是先想好完整答案。

再把答案一次性说出来。

🔍 它会根据当前上下文，一步步预测下一个 `Token`。

`Token` 是模型处理文本时的基本小块。

一个回答看起来是一整段。

实际由很多次“预测下一个小文本块”拼成。

📚 这不代表 `LLM` 没有能力。

大规模训练让它学到语言模式、知识关联和任务格式。

所以它能写出很像解释、推理和总结的内容。

但流畅表达不等于事实可靠。

涉及数据、医学、法律、金融和重要决策时，

一定要核查来源。

真正需要记住的是：

`LLM` 很会生成“看起来合理的下一步”。

但合理不等于真实。
