# 资料核查

## 核查日期

2026-06-13

## 核查来源

1. OpenAI API 文档：Using tools  
   https://developers.openai.com/api/docs/guides/tools

   核查要点：
   - OpenAI 文档把工具用于扩展模型能力，例如内置工具、`function calling`（函数调用）、`tool search`、远程 `MCP` 等。
   - 文档中的具体模型、参数和工具类型会随平台更新，本稿不使用这些细节作为核心表达。

2. Anthropic Claude API 文档：Tool use with Claude  
   https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview

   核查要点：
   - `Tool use` 让模型调用用户定义或平台提供的函数/工具。
   - 模型会基于用户请求和工具说明决定是否调用工具，并返回结构化调用；执行位置可能在应用侧，也可能在平台服务端。

3. Model Context Protocol 文档：Tools  
   https://modelcontextprotocol.io/specification/2025-06-18/server/tools

   核查要点：
   - MCP 中的 `Tools` 让模型与外部系统交互，例如查询数据库、调用 API 或执行计算。
   - MCP 文档强调工具调用涉及信任、安全和人工确认边界。

## 本篇采用的科普表达

- 将 `Tools` 解释为“AI 系统开放给模型的一组外部能力入口”。
- 强调模型通常负责选择和发起结构化调用，真正执行由应用、后端服务或平台运行环境完成。
- 用“模型判断、工具提供能力、系统执行和控制边界”解释整体关系。
- 区分 `Tools` 和 `MCP`：前者是具体可调用能力，后者是连接这些能力的一类标准方式。

## 需要避免的说法

- “有了 Tools，模型就自己会操作所有软件”：不准确。
- “工具调用一定比模型直接回答更可靠”：不准确，仍取决于工具质量、参数、权限、数据源和校验。
- “Tools 等于 MCP”：不准确。`Tools` 是可调用能力，`MCP` 是连接和暴露外部能力的一套协议。
- “Function Calling、Tool Use、MCP Tools 在所有平台完全等价”：不准确。不同平台实现、字段和执行方式会有差异。
